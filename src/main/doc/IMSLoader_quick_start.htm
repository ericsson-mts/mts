1        Overview

IMSLoader main features are the following:

·         The product is an Internet Protocol (IP) multi-protocol tester at layer 3; so we are able to support only IP based protocols.

·         IMSLoader has been designed specially for IMS architecture and we of course manage the main IMS protocols : SIP, RTP, Diameter, HTTP

·         Moreover it also supports a lot of other application protocols like Radius, RTSP, SMTP, IMAP, POP, SMPP, UCP, SIGTRAN, MSRP, SNMP  developed on customer demand.

·         We finally provide the support of the 4 main transport protocols: UDP, TCP, TLS and SCTP.

·         The tests are described in XML files with a test and many scenarios input files

·         With the IMSLoader product, you are able to perform

o       Protocol, functional, non-regression tests (sequential mode)

o       Load endurance and stress tests (load mode)

o       Simulation of equipments as the client, server or both sides

·         Users acts with 2 different interfaces :

o       A graphical one (for test writing and tuning)

o       A command-line (for load or automatic testing)

·         The product is easy to use because of logging management and rich statistics presentation.

·         IMSLoader can be active as a simulator (send and receive messages) for test-bed network hosts or as a network analyzer (capture messages) for network monitoring on production platforms.

·         Our solution is a software application entirely written in java and so you can run it on a lot of commercial platforms (support on Windows, Linux platforms and so on). There is no need to buy expensive hardware.

·         The application is modular and so split in different modules; then the customer can buy only what he needs.

·         The product documentation provided with the application is very complete: quick start, features description, demonstration release note and user manuals.

·         As examples, we furnish some IMSLoader XML scripts to illustrate the main features of the product. You could use them to learn the product use or as a starting point to develop new scripts: you only need to customize or configure them.

·         You have the possibility to run a test remotely with the master/slave feature on different machine (scalability feature)

·         Finally IMSLoader is an Open Source product: there is no license control mechanism; so you are able to install it on every host you will need without any restriction.

2        Requirements and dependencies

Operating System

IMSLoader is officially supported on the following platforms:

* Windows XP (not available for SCTP protocol)

* Linux Mandriva

However, IMSLoader may work fine on any Java platform: all LINUX distribution, UNIX system like Solaris, all Windows systems. But for other platforms than the supported, please contact us to have more information.

 

Java Runtime Environment

IMSLoader needs the JRE 1.5 or later (http://java.sun.com/downloads/).

 

External dependencies

1)     If you are willing to use the SCTP protocol, this is only possible on LINUX platform and then you have first of all to install some external libraries: LKSCTP et javaSCTP. See the next chapter for installation.

2)     If you are willing to use the passive mode (capture probe), you have first of all to install some external libraries: JPCAP. See the next chapter for installation.

3        Installation procedure

As the setup software is itself in Java, the installation procedure is the same for all supported Operating Systems.

 

Install JRE 1.5 or later

Follow the instruction provided by the JDK installer.

 

Install the IMSLoader software

Execute the following shell commands:

java jar imsloader_<Version_Number>_setup_<Customer_Reference>.jar

NOTE: For windows, you could use on the jar file the context menu (mouse right button) <Open With/Java(TM) 2 Platform> or just double click on the jar setup file

On the following of the document, the variable <InstallDirectory> will refer to the installation directory you have chosen at this point.

 

Install the LKSCTP and javaSCTP library => SCTP protocol

Please refer to the CORE user manual to have the detailed procedure (chapter 2.1)

 

Install the JPCAP library => Passive capture mode

Please refer to the CORE user manual to have the detailed procedure (chapter 2.2)

 

Install the audiocomp tool => Audio G711 comparison

Please refer to the CORE user manual to have the detailed procedure (chapter 2.3)

 

4        Starting point

4.1     Read the documentation

Look at the following documentations:

After the installation, all the documentation is located at the doc path (in the <InstallDirectory>/doc directory). This directory contains the following documents:

* IMSLoader-Pres-technical.ppt: => a technical presentation of IMSLoader dealing with an overview of the product features.

* IMSLoader-Pres-demos.ppt: => some demonstrations of the IMSloader main use cases.

* IMSLoader-ReleaseNotes.txt: => the release note of the product describing new features and bug fixes.

* IMSLoader_user_manual_core.htm: => all the user manuals: this one is for the core of the application and there are others for all the modules of the application.

* IMSLoader_benchmark.htm: => a benchmark document dealing with the performance of the tool for each protocol.

4.2     How to start the application?

Start the IMSLoader GUI application

From the installation directory, execute the following shell commands from the <InstallDirectory>/bin location:

* cd <InstallDirectory>/bin

* startGui.bat <test file> (Windows environment)

or

* ./startGui.sh <test file> (Windows environment)

 

NOTE: on Linux platform specially, in case you will listen on system ports (number less than 1024) or you will create capture probe, then you need root privileges.

4.3     Open your first test

The tutorial directory located at <InstallDirectory>/tutorial path contains some examples of all the tool usages. It could be for you a good starting point if you start a new testing activity.

 

To open your first tests coming from the tutorial for example:

With the <File/Open> menu item, open a test from the tutorial directory at the root of the installation directory (test.xml file).

Each testcase defined in the test appears line by line e.g. for Core, test.xml and testcase are located under <InstallDirectory>/tutorial/core directory

4.4     Look at the input files (test and scenario)

To have a look at the input files, do the following actions:

* With the <File/Edit> button of a testcase to edit your test XML script of the test you have opened.

* Click on the <Edit> button of a testcase to edit the scenario XML scripts of your testcase.

4.5     Run your first tests (different modes)

To run testcase, you will have the 3 following possibilities:

* Click simply on the <Run> button of a testcase to run it manually.

* Click simply on the <Test/Run:sequential> menu item to run sequentially all the testcase which is selected. => functional, non-regression protocol test.

* Click simply on the <Test/Run:parallel> menu item to run in load mode (in parallel) all the testcase which is selected  => load, stress, benchmark test.

4.6     Show the output resulting data

To have a feedback on the test running, you will have 2 ways:

* Click on the <Logs> checkbox of a testcase, to see the scenarios logs in call-flow mode. Use the <Topics> menu item to see more logs information. => functional test.

* Click on the <Stats/Show real time> menu item to display the statistics panel => load test.

6        Tutorial content

User can consult a lot of scripts examples in the tutorial directory.

These scripts can be use as starting points for new script writing.

There is a directory for each protocol supported by IMSLoader. For major part of these types of directories, there is a directory with load tests for this protocol. 

There are some specials directories:

 

/demos/

Package of scripts for demonstrate quickly main possibilities of scripting with IMSLoader.

 

/core/

Package of scripts for learn how to use core functions (more information about core user manual).

 

/master/

Package of scripts for learn how to use master functions (more information about master user manual ).

 

/<ppp>/

Package of scripts for learn how to use protocol <ppp> functions (more information about <ppp> user manual ).

 

 

7        Uninstallation procedure

As the setup software is itself in Java, the uninstallation procedure is the same for all supported Operating Systems.

Uninstall the IMSLoader software

Execute the following shell commands:

java jar <InstallDirectory>/Uninstaller/uninstaller.jar

NOTE: For windows, you could used on the jar file the context menu (mouse right button) <Open With/Java(TM) 2 Platform> or just double click on the jar setup file

On the following of the document, the variable <InstallDirectory> will refer to the installation directory you have chosen at this point.

NOTE: Before to uninstall the application, dont forget that the <InstallDirectory> directory will be totally removed. So if you have some test files into it, then please backup them before starting the uninstallation.
